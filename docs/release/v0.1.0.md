# v0.1.0

Initial release of `tokenmaxxing-ai`, a portable approval-first workflow package for AI coding agents.

## Highlights

- Adds the `tokenmaxxing-ai` CLI installer.
- Installs `/simplify` workflows that find over-engineered code, present a plan, ask for approval, edit narrowly, and verify.
- Installs `/review` workflows with lite, mid, and ultra modes for security, correctness, tests, maintainability, and simplification opportunities.
- Adds adaptive `.tokenmaxxing.md` project memory so repeated review and simplification runs can preserve concise local guidance.
- Ships standalone Codex and Claude Code skills with bundled reference checklists.
- Ships rule or command adapters for Cursor, GitHub Copilot, OpenCode, CommandCode, Antigravity, and Kiro.

## Supported Targets

- Codex
- Claude Code
- Cursor
- GitHub Copilot
- OpenCode
- CommandCode
- Antigravity
- Kiro

## Install

After the npm package is published:

```bash
npx tokenmaxxing-ai install
```

From a checked-out repository:

```bash
node bin/tokenmaxxing-ai.js install --dry-run --target all
node bin/tokenmaxxing-ai.js install --yes --verify --target all
```

## Verification

The release candidate was prepared with:

```bash
npm test
npm run check
npm pack --dry-run
node bin/tokenmaxxing-ai.js install --dry-run --target all
node bin/tokenmaxxing-ai.js install --yes --verify --dir .tmp-install --target all
TOKENMAXXING_AI_HOME=.tmp-home node bin/tokenmaxxing-ai.js install --yes --verify --dir .tmp-install-user --scope user --target codex
```

## Security Notes

- Installer path resolution rejects absolute adapter paths and traversal outside the selected install root.
- Generated workflows require explicit approval before edits.
- CI token permissions are read-only for repository contents.
